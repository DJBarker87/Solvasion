# Solvasion — Blog Summary

Solvasion is a fully on-chain territory conquest game built on Solana where players battle over a hexagonal grid overlaid on a real map of Europe. What makes it unique is the cryptography: defence allocations are hidden using Pedersen commitments on the Ristretto255 elliptic curve, meaning you can see the entire map and who owns what, but you never know how strong any position really is until combat forces a reveal. Every game action — claiming territory, garrisoning hexes, launching attacks, revealing defences — is a Solana transaction, with the on-chain program as the single source of truth. Seasons run up to 28 days, progressing through land rush, open warfare, and escalation phases. It's free to play with no token and no entry fees — players only pay refundable Solana rent deposits and negligible transaction fees.

The tech stack spans three layers: a 27-instruction Anchor/Rust program handling all game logic and Pedersen verification via Solana's native curve25519 syscalls (benchmarked at just 5,000 compute units — cheaper than SHA256); a Node.js backend built on Fastify and SQLite that indexes on-chain events, runs a reconciliation crank, manages three NPC bots with distinct personalities, and provides a Guardian auto-reveal service for offline players; and a React frontend using Mapbox GL JS with H3 hexagonal overlays, client-side Ristretto255 cryptography via @noble/curves, real-time WebSocket updates, and wallet integration via Solana wallet-adapter. The map covers Western Europe at H3 resolution 3 (251 hexes for Season 1), with bridge edges connecting islands across sea crossings. The entire project — program, backend, frontend, and 33 passing integration tests — was built by a maths teacher using Claude as the sole development tool.
